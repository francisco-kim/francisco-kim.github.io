* How it works
The site is written in [[https://orgmode.org][Emacs Org-mode]] and exported to static HTML.

- =src/= holds the source: =index.org= and =cryptograms.org= are the pages; =*.suborg=
  files (e.g. =fuga_1.suborg=, =ToC.suborg=) are fragments pulled in with =#+INCLUDE=;
  =src/assets/= holds the images, audio, and PDFs.
- =build.sh= empties =docs/= and runs =emacs -Q --script build-site.el=.
- =build-site.el= installs its only dependency, =htmlize= (kept under =.packages/=), and
  exports everything in =src/= into =docs/=.
- =docs/= is the generated output that GitHub Pages serves.

Important: the HTML export uses *only Emacs* — it does not run any Python. The
Python/=music21= analyses are pre-computed and their results (figures, tables) are committed,
so the site can always be rebuilt from any machine that has Emacs. See
[[Regenerating the analyses]] for when Python is actually needed.

* Publishing (the procedure)
** Option A — push and let CI publish (the normal way)
A push to the =main= branch triggers the GitHub Action in
=.github/workflows/publish.yml=, which:

1. checks out the repository,
2. installs a pinned version of Emacs,
3. runs =./build.sh=,
4. deploys the freshly built =docs/= to the =gh-pages= branch, which is what the live site
   serves.

So the live site always reflects what CI rebuilds from =src/= — committing =docs/= yourself
is only a local convenience.

#+BEGIN_SRC sh
git add -A
git commit -m "Update site"
git push origin main
#+END_SRC

** Option B — build locally to preview before pushing
#+BEGIN_SRC sh
./build.sh                 # regenerate docs/ from src/
xdg-open docs/index.html   # open the result in a browser (or just double-click it)
#+END_SRC

This needs only Emacs installed; nothing else.

* Regenerating the analyses
You only need Python/=music21= when you want to *re-run an analysis to produce a new figure
or result* — not to publish the site.

The active code blocks in =src/fuga_1.suborg= are marked =:eval never-export= and show
committed results. Just above each one sits the original interactive block
(=#+BEGIN_SRC ipython :session fugue ...=). To regenerate, evaluate that interactive block
in Emacs (via emacs-jupyter / Jupyter), commit the new image, and leave the
=:eval never-export= block in place.

The Python dependencies are pinned in =requirements.txt=:

#+BEGIN_SRC sh
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
#+END_SRC

Rendering scores with =music21= additionally requires a local MuseScore (MuseScore 4)
install configured in =music21=.
