Webpage showing the compositional ideas behind my Fuga I and other music-related
personal notes.

* How it works
The site is written in [[https://orgmode.org][Emacs Org-mode]] and exported to static HTML.

- =src/= holds the source. =index.org= is the home page (a welcome and the featured
  pieces); each composition gets its own page (e.g. =fuga_1.org=), plus there is
  =cryptograms.org=. The actual prose/analysis lives in =*.suborg= fragments (e.g.
  =fuga_1.suborg=) pulled in with =#+INCLUDE=. =common.setup= is the shared page header
  (author, stylesheet link, options) so pages don't repeat it. The look of the site is
  one hand-written stylesheet, =src/assets/css/site.css=; the navigation header and
  footer on every page come from =org-html-preamble= / =org-html-postamble= in
  =build-site.el=. =src/assets/= holds the images, audio, and PDFs.
- =build.sh= empties =docs/= and runs =emacs -Q --script build-site.el=.
- =build-site.el= installs its only dependency, =htmlize= (kept under =.packages/=), and
  exports everything in =src/= into =docs/=.
- =docs/= is the generated output that GitHub Pages serves.

Important: the HTML export uses *only Emacs* — it does not run any Python. The
Python/=music21= analyses are pre-computed and their results (figures, tables) are committed,
so the site can always be rebuilt from any machine that has Emacs. See
[[Regenerating the analyses]] for when Python is actually needed.

* Adding a new composition
Each composition is one page. To add one (say =prelude=):

1. Write the prose/analysis in =src/prelude.suborg= (a fragment, no header needed). Put any
   images/audio under =src/assets/=.
2. Create the page =src/prelude.org= by copying =fuga_1.org=. The minimal header is just:
   #+BEGIN_SRC org
   ,#+TITLE: My Prelude
   ,#+DESCRIPTION: A short description.
   ,#+SETUPFILE: common.setup
   #+END_SRC
   followed by =#+INCLUDE: prelude.suborg :minlevel 1=.
3. Add the piece to the index in =src/compositions.org=, in two places (copy the Fuga I
   entry as a template for both):
   - an entry section: a short blurb, the audio player, and the
     "Read the full analysis …" button (=<a class="cta" href="prelude.html">=);
   - one =<li><a href="prelude.html">My Prelude</a></li>= line in the sidebar list at the
     top of the file (the =#+BEGIN_EXPORT html= block), which is what shows in the left
     bar on wide screens.
   The navigation header (defined by =org-html-preamble= in =build-site.el=) stays fixed
   at Home · Compositions · Cryptograms, so it does not need editing.
4. Optional: to highlight it on the landing page, add a short featured block to =index.org=
   (see the "Featured: Fuga I" section there as a template).
5. Run =./build.sh= and open =docs/prelude.html= to check it.

* Publishing (the procedure)
** Option A — push and let CI publish (the normal way)
A push to the =main= branch triggers the GitHub Action in
=.github/workflows/publish.yml=, which:

1. checks out the repository,
2. installs a pinned version of Emacs,
3. runs =./build.sh=,
4. deploys the freshly built =docs/= to the =gh-pages= branch, which is what the live site
   serves.

So the live site always reflects what CI rebuilds from =src/= — committing =docs/= yourself
is only a local convenience.

#+BEGIN_SRC sh
git add -A
git commit -m "Update site"
git push origin main
#+END_SRC

** Option B — build locally to preview before pushing
#+BEGIN_SRC sh
./build.sh                 # regenerate docs/ from src/
xdg-open docs/index.html   # open the result in a browser (or just double-click it)
#+END_SRC

This needs only Emacs installed; nothing else.

* Regenerating the analyses
You only need Python/=music21= when you want to *re-run an analysis to produce a new figure
or result* — not to publish the site.

The active code blocks in =src/fuga_1.suborg= are marked =:eval never-export= and show
committed results. Just above each one sits the original interactive block
(=#+BEGIN_SRC ipython :session fugue ...=). To regenerate, evaluate that interactive block
in Emacs (via emacs-jupyter / Jupyter), commit the new image, and leave the
=:eval never-export= block in place.

The Python dependencies are pinned in =requirements.txt=:

#+BEGIN_SRC sh
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
#+END_SRC

Rendering scores with =music21= additionally requires a local MuseScore (MuseScore 4)
install configured in =music21=.
